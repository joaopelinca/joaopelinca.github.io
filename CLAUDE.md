## Rubric Alignment (KNES 381 — 100 pts)

### 1. Git & GitHub Workflow (15%)
- Commit **frequently** with descriptive messages (e.g., `add: BMI calculator python script`, `fix: navbar active state on mobile`)
- Never batch all work into one session — commit after each meaningful change
- `README.md` must explain: project purpose, tools used, how to navigate the site

### 2. Website / HTML / Markdown Structure (15%)
- Use proper Markdown hierarchy: `#`, `##`, `###` — never skip levels
- All pages need front matter: `title`, `layout`, `permalink`, `icon`
- Navigation must be clear and all internal links must work
- Use Chirpy's built-in components (tabs, callouts, code blocks) for polish

### 3. Computational Tool — Python (30%) ← HIGHEST WEIGHT
- Build a **fully functional Python script or notebook** processing kinesiology data
- The BMI calculator is the anchor — but extend it: add BMR, VO2 max estimate, or fitness classification
- Inputs must dynamically affect outputs (not hardcoded)
- All code must be **documented with comments**
- Consider a Jupyter notebook that can be linked or embedded (via nbviewer or GitHub render)

### 4. Programmatic Data Visualization (20%)
- Charts must be **generated by Python code** (matplotlib/seaborn), not manually created in Excel/Canva
- Charts must **update when data changes** — demonstrate this in code comments or a notebook
- Suggested charts: BMI distribution, VO2 max norms comparison, heart rate zones, force-velocity curve
- Export as high-res WebP/PNG and embed in the relevant site pages
- Label axes clearly, use professional color palettes, include titles and legends

### 5. Additional Technical Component (10%)
- This must be **distinct** from the Python analysis and charts above
- Good options:
  - A secondary Python feature (e.g., a pace/training load calculator)
  - An Excel system with formulas for a kinesiology dataset
  - A Scratch logic program (already done in class — link it)
- MATLAB work from sensorimotor neuroscience course can go here as a bonus showcase

### 6. Written Communication & Site Usability (10%)
- Every page needs an introduction paragraph — no orphan headers
- Proofread all text before final submission
- All links (internal + external) must be tested and functional
- Writing tone: professional but approachable (portfolio audience = future employers + professors)

## Git Workflow Rules

```bash
# Good commit message format:
git commit -m "add: about page with bio and skills section"
git commit -m "feat: BMI calculator python script with matplotlib chart"
git commit -m "fix: broken link in projects page"
git commit -m "style: update profile photo and sidebar avatar"
git commit -m "docs: update README with site navigation guide"
```

- Commit **after every logical unit of work** — not just at the end of sessions
- Push to GitHub regularly so the timestamp history is authentic (it's graded!)
- Branch for experimental features: `git checkout -b feature/bmi-calculator`

## Python Script Standards

## File Naming Conventions

- Never use spaces in any filename
